# Circuit Plan

**Week:** Assigned Week 3, Due Week 4
**Due:** Feb 23
**Weight:** Part of Electronic Circuit & Microcontroller (10%)

## Learning Outcomes

This assignment maps to the following learning outcomes:

- **LO 2 - Tools, Materials, Methods:** Write and debug basic Python code and create electronic circuits using microprocessors.

## Description

Design and document a basic electronic circuit plan that will be part of your emergent object. This assignment demonstrates your understanding of electricity fundamentals and circuit design principles.

## Requirements

- Create a circuit diagram showing:
  - Power source
  - At least one input component (button, sensor, etc.)
  - At least one output component (LED, speaker, motor, etc.)
  - Proper connections and polarity
  - Component values (resistors, etc.)
- Include a written explanation of how the circuit works
- List all required components with specifications
- Document expected behavior

## Deliverables

- Circuit diagram (hand-drawn or digital)
- Components list with specifications
- Written explanation (200-300 words) describing:
  - Circuit functionality
  - How components interact
  - Safety considerations
- Upload to Canvas

### Technical Requirements Checklist
- [ ] Power source specified with voltage
- [ ] At least one input component
- [ ] At least one output component
- [ ] Proper polarity indicated
- [ ] Resistor values calculated (show work)
- [ ] All connections clearly labeled
- [ ] Ground connections shown
- [ ] Component specifications listed

### Safety Considerations
- Resistors properly calculated to protect LEDs
- Correct polarity for all components
- Appropriate voltage levels
- No short circuits
- Components within safe operating ranges

### Assessment Focus
- **Technical accuracy:** Correct application of Ohm's Law and circuit principles
- **Safety:** Proper component protection and safe voltages
- **Clarity:** Easy to understand and build from your documentation
- **Preparation:** This plan should guide your actual circuit building

## Tips

- Start with a simple circuit and test it on a breadboard
- Calculate resistor values properly to protect components
- Double-check polarity for LEDs and power connections
- Label all components clearly on your diagram
- Consider what kind of interaction you want in your final object
- Use proper schematic symbols or clearly label hand-drawn diagrams
