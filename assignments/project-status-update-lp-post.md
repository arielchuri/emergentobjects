# Project Status Update Learning Portfolio Post

**Week:** Assigned Week 9, Due Week 10
**Due:** Apr 13
**Weight:** Part of Learning Portfolio engagement (10%)

## Learning Outcomes

This assignment maps to the following learning outcomes:

- **LO 7 - Reflection:** Document their process and reflect on their learning via the Parsons Learning Portfolio.

## Description

Create a comprehensive status update documenting your progress on the final emergent object project. This post should show where you are in the process, what's working, what challenges you're facing, and your plan for completion.

## Requirements

- Document current project status including:
  - What's complete and working
  - What's in progress
  - What still needs to be done
  - Current challenges and roadblocks
  - Solutions you're exploring
  - Timeline for completion
- Include visual documentation (photos, videos, screenshots)
- Reflect on your learning and process (200-300 words)
- Update project goals if needed

## Deliverables

- Complete Learning Portfolio post with:
  - Status report with checklist or progress indicators
  - Visual documentation
  - Written reflection (200-300 words)
  - Realistic timeline for remaining work

### Purpose of This Assignment
This status update serves multiple functions:
- **Self-assessment:** Honest evaluation of where you are
- **Planning tool:** Creating roadmap for completion
- **Communication:** Letting instructor know if you need support
- **Documentation:** Recording process for portfolio
- **Accountability:** Keeping yourself on track

### Being Honest About Status
The most valuable status updates are **honest**:
- If you're behind, say so - we can help
- If you're stuck, explain the problem - we can troubleshoot
- If you need to adjust scope, propose it - we can discuss
- If you're ahead, great - show what you're doing next

### Red Flags vs. Green Lights
**Concerns (seek help):**
- No working prototype yet
- Major technical problems unsolved
- Significant scope/timeline issues
- Haven't started enclosure
- Code not functioning

**Good signs:**
- Core functionality working
- Challenges identified with solutions in progress
- Clear next steps
- Enclosure in progress
- On track for completion

### Assessment Focus
- **Honest self-evaluation:** Being real about status
- **Problem-solving mindset:** Actively working through challenges
- **Planning skills:** Creating realistic completion strategy
- **Communication:** Clearly conveying status and needs

## Tips

- Be honest about where you are - this helps you get support
- Identify specific, actionable next steps
- Document problems as well as successes
- Show your work in progress, even if messy
- Consider what help or resources you might need
- Break down remaining work into small, manageable tasks
- This is a checkpoint to help ensure you finish strong
- Use this opportunity to ask questions or request help
